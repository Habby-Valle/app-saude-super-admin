# Skill: Write a PRD - App Saúde

Sempre siga esta estrutura:

1. Visão Geral
2. Objetivos do Produto
3. Personas
4. Funcionalidades Principais (separadas por Must / Should / Could)
5. Requisitos Não Funcionais (segurança, performance, LGPD, multi-tenant)
6. Priorização
7. Dependências Técnicas

Ao final, pergunte se deseja prosseguir para a Tech Spec.
Salve sempre como PRD.md na raiz ou em docs/.
